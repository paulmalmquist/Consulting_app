## Post 1 — Market Insight

**Headline:** PropTech Investors Are Betting on Ownership, Not Features

PropTech investment hit $16.7B in 2025. January 2026 alone pulled in $1.7B — a 176% jump year over year. Four new AI-native unicorns since last summer, all focused on automating leasing, lending, and valuation workflows.

That is a lot of capital flowing into a market that is still figuring out what it actually wants to buy.

The pattern worth watching: most of this investment is going into point solutions. One tool for valuation. One for leasing. One for lending. Each with its own data model, its own interface, and its own contract renewal cycle.

That approach works until it doesn't. When a vendor changes its pricing, its API, or its roadmap, the workflow that depended on it breaks. The firm didn't own the workflow. It rented it.

The four unicorns getting funded right now are proving AI can automate real estate operations at scale. What they haven't solved is where that automation lives. The firms that figure out systems ownership, not just workflow automation, will be the ones that compound the value over time.

## Post 2 — Systems Architecture Commentary

**Headline:** When Your Vendor Adopts an Agent Framework, You Still Don't Own the Agent

SAP. Salesforce. ServiceNow. Siemens. Adobe. All of them just adopted NVIDIA's Agent Toolkit.

That is a meaningful signal. Enterprise software incumbents are acknowledging that agentic execution is the next layer. Their bet is that bolting an agent framework onto an existing SaaS platform is how they stay relevant.

Here is the architectural problem with that approach.

An agent running inside a vendor's platform inherits the vendor's constraints. It can only see the data the platform exposes. It can only take actions the platform permits. The authorization model, the audit trail, the escalation logic — those all live on the vendor's infrastructure, not the firm's.

Firms running agent frameworks this way are adding automation without adding ownership. The workflow becomes more capable. It also becomes more fragile, because the capability is rented.

The alternative is building agent execution inside a system the firm controls. The authorization model is yours. The audit trail is yours. The data the agent can see is defined by your policy, not your vendor's API surface.

That is not a small architectural distinction. It is the difference between a firm that operates through its software and a firm that operates through its vendor.

**Hook:** SAP, Salesforce, ServiceNow, Siemens, and Adobe all adopted NVIDIA's Agent Toolkit this week.

## Post 3 — Tactical Automation Example

**Headline:** The LP Report Shouldn't Take Three Days

A REPE firm closes a deal. The LP report is due in 10 days. The associate pulls the model from the deal folder, updates the waterfall manually, formats the distribution schedule in Excel, and emails it to IR for review.

IR has questions. The model changes. The waterfall updates. The formatted report goes stale. Three revision cycles later, the report goes out on day nine.

In an execution environment, the deal close triggers a structured workflow. The waterfall runs off the canonical deal model. The LP report pulls current data, formats to the firm's template, and flags any variance from the prior period. The associate reviews a completed draft, not a blank page.

The report takes two hours instead of three days. The associate spends the rest of the week on the next deal.

The work didn't get easier because of AI. It got easier because the workflow is owned by the firm, not assembled from scratch each time.

**Hook:** The LP report shouldn't take three days.
