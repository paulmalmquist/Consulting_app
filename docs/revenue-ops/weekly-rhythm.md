# Weekly Operating Rhythm

Designed for lean execution (one person + autonomous support). Every day has a theme that compounds toward first revenue.

## Monday: Pipeline + Target Review
**Time block:** 2-3 hours morning

1. Open Novendor consulting pipeline view
2. Review weekend sales signal output (docs/sales-signals/)
3. Score and prioritize any new leads from signals
4. Update all opportunity stages and next-step dates
5. Identify stuck deals (>7 days without activity) and define unblock action
6. Select 3-5 new target accounts to research and add to CRM
7. Review sales positioning output (docs/sales-positioning/) for messaging updates
8. Set the week's outreach targets

**Autonomous support:** pipeline-health-review (6 AM), target-account-discovery (8 AM), offer-refinement-check (7 AM), content-calendar-update (8 AM)

## Tuesday: Proof Asset + Offer Work
**Time block:** 3-4 hours focused work

1. Pick the highest-priority proof asset from the backlog
2. Build or refine that asset (aim for completion or significant progress)
3. Update offer positioning based on objections heard this week
4. Draft or refine proposal for any active opportunity
5. Review and improve diagnostic questionnaire if feedback received
6. Update proof-backlog.md status

**Autonomous support:** proof-asset-production (7 AM)

## Wednesday: Outbound + Follow-Up Push
**Time block:** 2-3 hours

1. Execute 10+ outreach touches across all channels
2. Follow up on all open conversations older than 3 days
3. Send personalized messages to warm leads
4. Log every touch in CRM (cro_outreach_log) with channel, sentiment, next step
5. Process any inbound replies and schedule discovery calls
6. Ask for referral introductions from warm contacts

**Autonomous support:** outreach-draft-batch (9 AM), follow-up-sequencing (10 AM)

## Thursday: Product Demo + Objection Handling
**Time block:** 2-3 hours

1. Run any scheduled demo/discovery sessions
2. If no sessions: practice and improve demo flow
3. Document every objection, feature request, and "can it do X?" from conversations
4. Log product feedback in structured format (docs/revenue-ops/product-feedback/)
5. Update Winston product backlog with sales-driven priorities
6. Improve specific demo flows based on real feedback

**Autonomous support:** demo-improvement (7 AM)

## Friday: Review + Reprioritize
**Time block:** 2-3 hours

1. Update scoreboard.md with current metrics
2. Review week's conversion signals (what worked, what didn't)
3. Analyze: Which outreach got replies? Which demo moments landed? Which objections repeated?
4. Reprioritize target accounts based on response signals
5. Reprioritize offers based on demand signals
6. Draft next week's outreach batch (Monday sends)
7. Write 1 LinkedIn post for Monday publishing
8. Update TASKS.md with next week's priorities

**Autonomous support:** crm-hygiene (6 AM), conversion-analysis (8 AM), product-feedback-synthesis (7 AM)

## Saturday-Sunday: Passive
- Autonomous tasks continue running (sales signals, competitor research, etc.)
- Content publishes on schedule
- No active outreach (respect people's weekends)
- Optional: Light research, reading competitor updates, refining strategy notes
