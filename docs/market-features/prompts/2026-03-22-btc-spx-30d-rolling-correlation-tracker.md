## Specification

### What It Does
Computes and stores the 30-day rolling Pearson correlation between BTC and SPX daily log returns on a daily cadence. Surfaces a historical correlation chart in the Winston Market Intelligence module. Fires an alert when correlation crosses zero in either direction (decoupling: positive → negative, or recoupling: negative → positive). The current correlation value feeds into the Multi-Asset Regime Classifier Dashboard as a crypto-regime input signal.

### Data Layer

**New Tables (additive only):**
```sql
-- Daily BTC-SPX rolling correlation timeseries
CREATE TABLE public.btc_spx_correlation (
  correlation_id   uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id        uuid,
  business_id      uuid,
  calculated_date  date NOT NULL,
  correlation_30d  numeric(8,6) NOT NULL,     -- Pearson r, range [-1, 1]
  btc_return_30d   numeric(10,6),             -- BTC 30-day cumulative log return
  spx_return_30d   numeric(10,6),             -- SPX 30-day cumulative log return
  zero_crossing    boolean NOT NULL DEFAULT false,  -- true if sign changed vs. prior day
  crossing_direction text,                    -- 'decoupling' (+ → -) or 'recoupling' (- → +)
  data_points_used int NOT NULL DEFAULT 30,   -- actual days of data in window
  metadata         jsonb NOT NULL DEFAULT '{}',
  created_at       timestamptz DEFAULT now(),
  UNIQUE (tenant_id, calculated_date)
);

CREATE INDEX idx_btc_spx_corr_date   ON public.btc_spx_correlation (calculated_date DESC);
CREATE INDEX idx_btc_spx_corr_tenant ON public.btc_spx_correlation (tenant_id, calculated_date DESC);

ALTER TABLE public.btc_spx_correlation ENABLE ROW LEVEL SECURITY;
CREATE POLICY "tenant_read_btc_spx_corr" ON public.btc_spx_correlation
  FOR SELECT USING (tenant_id = auth.uid() OR tenant_id IS NULL);
```

**Data Sources:**
- **BTC daily close:** `fact_market_timeseries` symbol `BTC-USD` — daily close prices (existing table, additive reads)
- **SPX daily close:** `fact_market_timeseries` symbol `^GSPC` — daily close prices (existing table, additive reads)
- **Refresh cadence:** Daily at 6:30 AM UTC (after `market_regime_engine.py` completes, allowing the correlation value to feed into that day's regime snapshot)

**Data Pipeline:**
```
fact_market_timeseries (BTC-USD, ^GSPC daily closes)
  → backend/app/services/btc_spx_correlation_service.py
      (compute 30-day rolling log returns → Pearson r → zero-crossing detection)
  → public.btc_spx_correlation (storage)
  → GET /api/v1/market/correlation/btc-spx (history, up to 365 days)
  → GET /api/v1/market/correlation/btc-spx/latest (single latest value)
  → BtcSpxCorrelationChart (frontend render)
  → regime_sentinel_service.py (reads latest value as crypto regime input)
```

### Backend

**New Service File(s):**
- `backend/app/services/btc_spx_correlation_service.py`
  - `compute_btc_spx_correlation(tenant_id: UUID | None, as_of_date: date | None = None) -> BtcSpxCorrelationRow`
    - Fetches last 35 trading days of BTC-USD and ^GSPC closes from `fact_market_timeseries`
    - Computes daily log returns: `ln(close_t / close_{t-1})`
    - Computes 30-day Pearson r between aligned return series
    - Detects zero-crossing vs. previous `btc_spx_correlation` row
    - Upserts into `btc_spx_correlation` for the calculation date
    - Returns `BtcSpxCorrelationRow` dataclass
  - `get_latest_correlation(tenant_id: UUID | None) -> BtcSpxCorrelationRow`
    - Returns most recent row from `btc_spx_correlation`
  - `get_correlation_history(tenant_id: UUID | None, days: int = 180) -> list[BtcSpxCorrelationRow]`
    - Returns up to `days` rows ordered by `calculated_date DESC`

**New Route(s):**
- `GET /api/v1/market/correlation/btc-spx/latest`
  - Request: `{ tenant_id?: uuid }` (query param)
  - Response:
    ```json
    {
      "calculated_date": "2026-03-22",
      "correlation_30d": 0.13,
      "btc_return_30d": -0.142,
      "spx_return_30d": -0.088,
      "zero_crossing": true,
      "crossing_direction": "recoupling",
      "data_points_used": 30
    }
    ```
- `GET /api/v1/market/correlation/btc-spx`
  - Request: `{ days?: int = 180, tenant_id?: uuid }`
  - Response:
    ```json

      "series": [
        {
          "calculated_date": "2026-03-22",

        }
      ],
      "latest_correlation": 0.13,
      "regime_signal": "recoupling — BTC re-correlating with equities (risk-off confirming)"
    }
    ```

### Frontend

**New Component:**
- Name: `BtcSpxCorrelationChart`
- Location: `repo-b/src/components/market/BtcSpxCorrelationChart.tsx`
- Props:
  ```typescript
  interface BtcSpxCorrelationChartProps {
    tenantId?: string;
    days?: number;        // default 180
    compact?: boolean;    // true = single line only, false = full with annotations
  }
  ```

**Visualization:**
- **Chart type:** Line chart (recharts `LineChart` or `ComposedChart`)
- **Library:** `recharts`
- **Primary line:** 30-day rolling Pearson r vs. date
- **Reference line:** Dashed horizontal line at `y=0` (the decoupling/recoupling threshold)
- **Zero-crossing markers:** Vertical dotted lines at dates where `zero_crossing=true`, colored green (decoupling) or red (recoupling)
- **Color coding:** Line color shifts from green (r < 0, uncorrelated/decoupled) to red (r > 0, correlated/risk-asset behavior) with a neutral band for |r| < 0.15
- **Interaction:**
  - Hover tooltip shows date, correlation value, BTC 30d return, SPX 30d return
  - Click on a zero-crossing marker shows a callout: "Recoupling event — BTC realigning with equities" or "Decoupling event — BTC diverging from equities"
- **Summary badge:** Small pill below chart showing current r value + regime signal text ("Recoupling — risk-off confirming" or "Decoupled — uncorrelated zone")

**Integration Point:**
- **Primary:** Market Intelligence → Crypto tab (new additive tab)
- **Secondary:** Feeds a single-value correlation badge into the `RegimeClassifierWidget` crypto signal breakdown row
- **Navigation:** Dashboard → Market Intelligence → Crypto tab (additive, new tab alongside Regime and Alerts tabs)
