### Why This Exists
During the March 22, 2026 RISK_OFF_DEFENSIVE regime classification, three explicit escalation thresholds were identified as RISK_OFF_PANIC trigger signals: VIX > 35, HY OAS > 400bps, and DXY > 105. Winston has no automated mechanism to detect when these thresholds are breached — analysts would only discover escalation retroactively on the next daily rotation cycle. A VIX spike from 27 to 36 mid-session would go entirely unnoticed until the following morning.

Winston could not proactively alert when regime escalation conditions were breached. All verticals (REPE, Credit, PDS) operate against stale regime state until the next scheduled run. There is no mechanism to promote the current regime label from RISK_OFF_DEFENSIVE to RISK_OFF_PANIC_WATCH when 2+ triggers fire simultaneously, and no in-app notification surface for cross-vertical escalation events.

## Specification

### What It Does
A sentinel polling loop runs every 15 minutes during market hours (09:00–17:00 ET, Monday–Friday). On each poll it evaluates three escalation conditions against the most recent values in `fact_market_timeseries`:

- **VIX Trigger:** VIX `^VIX` close > 35
- **HY OAS Trigger:** HY OAS (FRED `BAMLH0A0HYM2`) > 400 bps
- **DXY Trigger:** DXY (`DX-Y.NYB`) close > 105

On any single trigger breach: inserts a row into `regime_alerts`, streams an SSE event to connected clients.
On 2+ simultaneous trigger breaches: additionally updates the latest `market_regime_snapshot` regime_label to `stress` and fires cross-vertical notifications to REPE, Credit, and PDS advisory channels.

Users can acknowledge alerts via API and configure per-sentinel thresholds via a settings endpoint.

### Backend

**New Service File(s):**
- `backend/app/services/regime_sentinel_service.py`
  - `evaluate_sentinel_triggers(tenant_id: UUID | None) -> list[AlertFired]`
    - Reads latest VIX, HY OAS, DXY values from DB + FRED
    - Compares against thresholds in `regime_sentinel_config` (falls back to hardcoded defaults)
    - Returns list of any breached triggers with values
  - `fire_alerts(triggers: list[AlertFired], tenant_id: UUID | None) -> list[UUID]`
    - Inserts rows into `regime_alerts`
    - If `len(triggers) >= 2`, calls `promote_regime_to_stress(tenant_id)` — additive UPDATE on latest `market_regime_snapshot` row only
    - Returns list of inserted `alert_id` UUIDs
  - `promote_regime_to_stress(tenant_id: UUID | None) -> None`
    - UPDATEs only the latest row in `market_regime_snapshot` where `regime_label != 'stress'`
    - Additive — does not touch any other table
  - `get_latest_alerts(tenant_id: UUID | None, limit: int = 20) -> list[RegimeAlert]`
    - Reads from `regime_alerts` ordered by `fired_at DESC`
  - `acknowledge_alert(alert_id: UUID, acknowledged_by: str) -> None`
    - Sets `acknowledged=true`, `acknowledged_at=now()`, `acknowledged_by` on the row
  - `get_sentinel_config(tenant_id: UUID | None) -> list[SentinelConfig]`
    - Returns current thresholds from `regime_sentinel_config`
  - `update_sentinel_config(tenant_id: UUID | None, updates: list[ConfigUpdate]) -> None`
    - UPSERTS threshold rows in `regime_sentinel_config`

**New Route(s):**
- `GET /api/v1/market/alerts/latest`
  - Request: `{ tenant_id?: uuid, limit?: int = 20 }` (query params)
  - Response:

  - Request: `{ acknowledged_by: string }`
  - Response: `{ "status": "ok", "alert_id": "uuid" }`
- `GET /api/v1/market/sentinel/config`
  - Response: `{ "sentinels": [{ "sentinel_name": "vix", "threshold_value": 35.0, "enabled": true, "poll_interval_minutes": 15 }] }`
- `PUT /api/v1/market/sentinel/config`
  - Request: `{ "updates": [{ "sentinel_name": "vix", "threshold_value": 32.0 }] }`
  - Response: `{ "status": "ok", "updated": 1 }`
- `GET /api/v1/market/alerts/stream`
  - SSE endpoint — streams `AlertFired` events as `text/event-stream`
  - Event shape: `data: { "alert_id": "uuid", "trigger_name": "vix", "trigger_value": 36.2, "fired_at": "ISO8601" }\n\n`

### Cross-Vertical Hooks

- **→ REPE:** On 2+ trigger alert, an advisory note is appended to the REPE underwriting context: "ALERT: Regime escalation triggers active (VIX > 35, HY OAS > 400bps). Stress-test all cap rate assumptions. No existing REPE service code modified."
- **→ Credit:** On 2+ trigger alert, the credit decisioning context window receives an additive advisory flag: "Regime stress conditions active — apply conservative DTI ceiling." Passed as context only, never as a hard rule. No existing credit service code modified.
- **→ PDS:** Escalation event surfaces as a market conditions advisory in PDS pipeline context: "Macro stress signal active — review construction financing assumptions." No existing PDS service code modified.
