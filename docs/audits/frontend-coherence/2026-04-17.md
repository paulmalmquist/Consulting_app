# Winston Repo Audit & Execution Report — 2026-04-17

## 1. Executive Summary

**Biggest problems found**
- `HistoryRhymesTab` and its fallback arrays render 100% hardcoded mock data as polished intelligence UI. No call to `POST /api/v1/rhymes/match` — users see invented numbers labeled `LIVE`. Highest-leverage trust breaker on the platform.
- No in-env environment switcher. Admin must return to `/app` to jump between workspaces, losing context and time.
- "Capability unavailable" is expressed four different ways: `CapabilityUnavailable` (polished), `DomainPreviewState` (eyebrow), ad-hoc error banners (transient), silent empty states. Users can't tell "not enabled" from "preview" from "service down."
- Environment lifecycle (`is_active`, `repe_initialized`) exists in data but never renders in the `/app` picker — every env looks equally live.

**Biggest wins already in the repo**
- Public homepage `/` is actually good — `WinstonPublicHome` explains the multi-environment model and middleware correctly routes auth'd traffic to `/app`.
- `WorkspaceIdentityBar` + breadcrumbs surface the "Winston → Environment → Module" triad on every lab route.
- `environmentCatalog` gives every first-class env a glow color, family label, login copy — consistent branding hook already exists.
- `useDecisionEngine` hook + `/api/v1/decision-engine` proxy already fetch real data into `CommandCenterLayout` — the wiring skeleton exists.

**Biggest remaining risks** (not addressed this session)
- Backend wiring for `POST /match` + full pipeline (state vector, pgvector, multi-agent forecaster) — T3.1.
- Capability manifest is split across three registries (`environmentAuth`, `DepartmentRegistry`, `CapabilityRegistry`) — no single declarative map of "env → modules → capabilities." T2.3.
- `HistoryRhymesTab` remains orphaned code — its mock arrays still power the fallback path when `useDecisionEngine` errors.

- Brier calibration loop
- Weekly deep research loop

## 11. Tips for tips.md

Three reusable patterns learned this session have been appended to [docs/tips.md](../../tips.md):

1. **Preview → Seeded → Live banner pattern** for any AI/ML surface that renders before the full pipeline ships. Forces honest copy and prevents silent mock-as-real leakage.
2. **Context-preserving route resolution** (`resolveSwitchTarget`) for cross-tenant navigation. Never carry deep entity paths; prefer module preservation when target has the same primary module.
3. **Single-source state taxonomy** (`capability-state-taxonomy.ts`) driving both `CapabilityUnavailable` and `EnvLifecyclePill`. Same words, same tones, same telemetry attr across capability and lifecycle surfaces.

See [docs/tips.md](../../tips.md) for the full entries.
