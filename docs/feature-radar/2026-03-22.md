## Feature Ideas

### 1. Predictive Investor Communication Parsing

**Classification:** ENHANCEMENT
**Signal source:** Juniper Square AI-powered CRM launch (NLP + predictive modeling on emails, docs, investor comms)

Per CAPABILITY_INVENTORY.md, Winston already has: CRM core (client management, lead tracking, pipeline, engagement tracking), document pipeline (extraction engine with profiles and writeback, OCR parser, text extractor), and LP/Investor tools (LP summary, capital call management, investor statement PDF, IR draft assembly).

**Gap:** Winston's document pipeline extracts structured data from documents, but does not yet apply predictive NLP to inbound investor communications (emails, letters, meeting notes) to auto-surface sentiment, intent signals, or predicted next actions. Juniper Square is marketing exactly this — using NLP to turn unstructured investor touchpoints into CRM intelligence.

**Enhancement scope:** Add an investor communication classifier to the existing document pipeline that tags investor emails/notes with sentiment (positive/neutral/concerned), intent (capital call response, redemption signal, re-up interest), and recommended follow-up action. Wire output into existing CRM engagement tracking.

| Dimension | Score (1-5) |
|---|---|
| Signal strength | 4 — Direct competitor shipping this; $1.1B valuation company |
| Complexity | 3 — Extends existing extraction engine + CRM; needs prompt engineering for classifier |
| Differentiation | 3 — Matches competitor parity; differentiation comes from Winston's copilot integration (Juniper Square doesn't have a copilot layer) |

**Files likely involved:** `backend/app/services/extraction_engine.py`, `backend/app/services/crm*.py`, document pipeline skill, CRM routes

### 2. Real-Time Data Context Streaming for Dashboard Widgets

**Classification:** ENHANCEMENT
**Signal source:** IBM/Confluent acquisition — real-time data pipelines becoming table stakes for enterprise AI agents

Per CAPABILITY_INVENTORY.md, Winston already has: Dashboard Composer (38KB service), environment binding, query intent translation, and RAG system with vector indexing and semantic reranking.

**Gap:** Winston's dashboard composition handles query-time data fetching but doesn't support streaming/push-based data updates to widgets. The IBM/Confluent signal suggests enterprise buyers will increasingly expect dashboards that reflect live data without manual refresh.

**Enhancement scope:** Add a lightweight SSE (Server-Sent Events) or WebSocket layer to the dashboard composer that pushes widget data updates when underlying data changes. Start with a single high-value widget type (e.g., deal pipeline status or fund NAV) as proof of concept.

| Dimension | Score (1-5) |
|---|---|
| Signal strength | 3 — Infrastructure trend, not direct competitor feature |
| Complexity | 4 — Requires event infrastructure, WebSocket/SSE plumbing, frontend subscription model |
| Differentiation | 3 — Nice-to-have for demos; not yet a buyer requirement in REPE |

**Files likely involved:** `backend/app/services/dashboard_composer.py`, frontend dashboard components, potentially new event bus service
