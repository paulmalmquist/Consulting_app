## Context: GPT-5.4 as Competitive Forcing Function

OpenAI's GPT-5.4 (released March 5, 2026) has clarified this week as the **primary competitive threat** in the REPE/finance AI space. New details confirm: native computer-use agents scoring 75% on OSWorld-Verified (above the 72.4% human baseline), 1M-token context window, ChatGPT plugins for Excel and Google Sheets in beta, and direct integrations with FactSet, MSCI, Moody's, and Third Bridge. On spreadsheet modeling benchmarks calibrated to junior IB analyst tasks, it scores 87.3%.

The 3/22 morning radar did NOT address GPT-5.4's CRE workflow implications. The 3/22 noon scan addressed the Anthropic Marketplace and MCP extensibility. This scan addresses the GPT-5.4 gap specifically.

## New Ideas (Beyond Previous Radar)

### 1. ENHANCEMENT — Winston Context Window Upgrade: 1M Token Deal Room

**Market signal:** GPT-5.4's 1M-token context window enables "whole-deal ingestion" — feeding an entire deal's worth of documents (OM, rent rolls, financials, legal, correspondence) into a single context and querying across them without chunking artifacts. OpenAI is marketing this directly at CRE use cases: offering memorandum review, rent roll analysis, lease abstraction, due diligence Q&A.

**Per CAPABILITY_INVENTORY.md, Winston already has:** Document pipeline (extraction engine, OCR parser, text extractor, extraction profiles), RAG system (vector indexing, semantic reranking, PsychRAG), deal pipeline (Deal Radar, deal detail, deal scoring), and the AI gateway with token budget management.

**The gap:** Winston's RAG architecture chunks documents and retrieves relevant fragments. This is the right architecture for large corpora but introduces retrieval errors on multi-document cross-referencing ("does the rent roll match the OM rent assumptions?"). GPT-5.4's 1M context eliminates the chunking step for deal-sized document sets (typically 20-80 pages total). Winston should offer a **"Deal Room Mode"** — a specialized context mode that loads an entire deal package into a single context window and enables cross-document Q&A without RAG retrieval. This would be a concrete Winston differentiator since REPE buyers will start asking about this capability after seeing GPT-5.4 demos.

**Enhancement scope:** Add a "Deal Room" context mode to the Winston AI gateway that bypasses RAG for small document sets and loads all documents directly into context. UI: a toggle in the Context Panel ("Use Deal Room Mode") that activates when total token count is under 800K. Existing RAG path remains for large corpora.

**Files likely involved:** `backend/app/services/ai_gateway.py`, `backend/app/services/rag_indexer.py`, `repo-b/src/components/copilot/` (Context Panel), deal-related document pipeline services.

| Dimension | Score (1-5) |
|---|---|
| Signal strength | 5 — GPT-5.4 is shipping this; REPE buyers will see demos |
| Complexity | 3 — Gateway already handles context routing; needs token-budget gate and UI toggle |
| Differentiation | 4 — Turns a GPT-5.4 threat into a Winston feature via existing architecture |

**Priority score: HIGH**

### 2. ENHANCEMENT — Adaptive Thinking Budget for REPE Query Complexity

**Market signal:** GPT-5.4 and competing models now implement "adaptive thinking" — dynamically allocating more compute (reasoning tokens) to complex queries and less to simple lookups. On Winston, this matters because REPE queries span a huge complexity range: "What's the current cap rate on Asset X?" (simple lookup) vs. "Model the IRR impact of a 150bps cap rate expansion across our 3 retail assets under a 5-year hold assumption" (multi-step financial reasoning).

**Per CAPABILITY_INVENTORY.md, Winston already has:** AI gateway with token budget management, model dispatch (routing), financial modeling (IRR timeline, Monte Carlo, scenario engine v1+v2), and assistant scope (context isolation).

**The gap:** Winston's AI gateway has a token budget manager but it appears to apply a fixed budget rather than adapting to query complexity. Per CAPABILITY_INVENTORY.md, the gateway is 189KB — large enough that this is likely already partially implemented, but it's not documented as "adaptive." Adding an explicit complexity classifier at the gateway layer (simple lookup → fast path, multi-step financial reasoning → extended thinking budget) would improve response quality on hard queries and reduce cost on easy ones simultaneously. This is the same architecture GPT-5.4 is marketing as a headline feature.

**Enhancement scope:** Add a query complexity classifier at the gateway entry point that assigns one of three tiers: FAST (simple lookups, < 2K thinking tokens), STANDARD (analytical queries, 8K thinking tokens), DEEP (multi-step financial modeling, 32K thinking tokens). Wire tier into existing model dispatch and token budget manager.

**Files likely involved:** `backend/app/services/ai_gateway.py`, `backend/app/services/assistant_blocks.py`

| Dimension | Score (1-5) |
|---|---|
| Signal strength | 5 — All major models shipping this; becomes buyer expectation |
| Complexity | 2 — Gateway already has routing and budget; needs classifier + tier config |
| Differentiation | 3 — Matches GPT-5.4 parity; Winston's REPE-specific tier calibration is the differentiator |

**Priority score: MEDIUM-HIGH**

### 3. ENHANCEMENT — Spreadsheet Modeling Agent: Counter to GPT-5.4 Excel Plugins

**Market signal:** OpenAI launched ChatGPT plugins for Excel and Google Sheets in beta alongside FactSet, MSCI, and Moody's integrations. On spreadsheet modeling benchmarks (junior IB analyst tasks), GPT-5.4 scores 87.3%. This is a direct competitive threat to Winston's Excel-touchpoint strategy.

**Per CAPABILITY_INVENTORY.md, Winston already has:** Excel add-in (`excel-addin/`), demo lab environments (32 types), capital projects service, draw management, and financial modeling (IRR, Monte Carlo, scenarios, amortization). The Excel add-in already exists.

**The gap:** Winston's Excel add-in exists but its current capabilities relative to GPT-5.4's new Excel plugins are unclear. The competitive risk is that a REPE analyst starts using ChatGPT's Excel plugin (connected to FactSet/MSCI data) instead of Winston's add-in. Winston's counter-positioning: the Excel add-in should be able to pull Winston's proprietary portfolio data (fund NAV, asset financials, deal metrics) directly into Excel — something OpenAI's generic plugin cannot do. The add-in needs a "push to Excel" action that exports any Winston table/model result directly into a formatted Excel sheet, and a "pull from Winston" action that lets the analyst reference live Winston data in Excel formulas.

**Enhancement scope:** Add bidirectional sync to the Excel add-in: (1) Export any Winston analytics result (table, scenario, waterfall) to a formatted Excel range with one click. (2) Enable a custom Excel function `=WINSTON(metric, asset_id, period)` that calls the Winston API and returns live data into a cell. Position against GPT-5.4 as: "GPT-5.4 connects to public market data. Winston connects to YOUR portfolio."

**Files likely involved:** `excel-addin/` (existing), `repo-b/src/app/api/v1/` (new Excel data endpoint), financial modeling services.

| Dimension | Score (1-5) |
|---|---|
| Signal strength | 5 — GPT-5.4 Excel plugin is shipping now; direct threat |
| Complexity | 3 — Add-in exists; needs bidirectional sync and custom function |
| Differentiation | 5 — Proprietary portfolio data in Excel is unmatched by any generic AI plugin |

**Priority score: HIGH**

New ideas beyond morning radar: **3**
