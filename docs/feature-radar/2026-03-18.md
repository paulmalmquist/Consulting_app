### 1. LP Reporting Autopilot

**Market signal:** Private equity firms are compressing LP reporting timelines from weeks to minutes using agentic frameworks. Allvue launched an AI assistant for capital calls, equity setup, and reporting. One megacap firm's IR team reported a 70% increase in time spent on direct LP engagement after deploying an agentic reporting stack. LP expectations are shifting — transparency, speed, and proof of digital competence are now central fundraising requirements. *(PYMNTS, EY, Acquis Consulting — March 2026)*

**Feature idea:** Winston LP Reporting Autopilot — a scheduled agent that pulls fund-level KPIs (IRR, equity multiple, DPI, TVPI), cross-checks actuals vs. underwriting, generates a draft LP narrative in configurable tone/detail level, and emails or exports a PDF packet per LP's stated preferences. Includes a "review before send" approval gate in the Winston UI before dispatch.

**Which module:** Fund portfolio page + LP management module + document pipeline. Needs a new `lp_report_runs` table and a PDF generation step downstream of the existing MCP tools.

**Enterprise value:** A REPE CFO running 5 funds and 40+ LPs spends 3–5 days per quarter assembling reports. Autopilot compresses that to a review-and-approve workflow. Directly supports fundraising velocity by demonstrating digital competence to LPs.

**Implementation complexity:** High — requires PDF templating, per-LP preference storage, email dispatch, and an approval UI. However, the data layer (fund KPIs, waterfall) is largely already in Winston.

**Priority score:** 9

**Priority rationale:** Strong competitive pressure from purpose-built LP portals (Juniper Square, Allvue). Winston already has the underlying fund data — this feature converts it into a differentiated workflow that no generic AI assistant can replicate. High enterprise impact, real fundraising leverage.

### 2. MCP Audit Trail & Tool Governance Dashboard

**Market signal:** Enterprise MCP adoption is hitting a wall on governance. Security researchers flagged prompt injection, tool permission sprawl, and "lookalike tool" attacks. The 2026 MCP roadmap explicitly targets audit trails, SSO-integrated auth, and gateway behavior as the top blockers for enterprise readiness. Entro launched an agentic governance product today (March 18) specifically offering MCP activity visibility and policy enforcement. *(CData, Mirantis, Entro — March 2026)*

**Feature idea:** Winston MCP Governance Dashboard — a UI surface at `/app/admin/mcp` that shows: (1) per-tool invocation logs with timestamp, user, prompt context, and result; (2) a policy layer where admins can restrict which tools are callable by role or environment; (3) anomaly flagging when tool chains match known exfiltration patterns. Exposed as read-only to REPE CFOs, write-access to Winston admins.

**Which module:** `backend/app/mcp/` — extend the existing 83-tool registry to emit structured audit events. New `mcp_audit_log` table. New frontend page in the app shell.

**Enterprise value:** REPE firms handle NDA-gated deal data, LP PII, and unreported fund performance. A CFO signing off on Winston as the firm's AI stack needs to see exactly what the AI did and why. This converts Winston from a black box into an auditable system of record — a requirement for enterprise procurement approval.

**Implementation complexity:** Medium — the MCP server already processes tool calls; adding structured event emission is additive. The UI is new but can reuse existing table/log components.

**Priority score:** 8

**Priority rationale:** Directly unblocks enterprise sales. Governance is table stakes for any CFO approval — without it, Winston cannot win multi-seat firm deals. The MCP registry is already Winston's unique asset; auditing it is a natural extension. Medium complexity, high unlock value.

### 3. Agentic Deal Radar — Autonomous Comp Sourcing

**Market signal:** Agentic AI is shifting from assistive to goal-driven in 2026 — systems that can receive a task in natural language, gather data from multiple sources, and iterate to a conclusion without hand-holding. In proptech, agentic tools are being deployed for procurement, pricing, and event programming. ICSC specifically called out agentic AI as "the next major phase of proptech." *(ICSC, CloudKeeper, Deloitte — 2026)*

**Feature idea:** Extend Deal Radar with an autonomous comp-sourcing agent. A user defines a deal type, submarket, and vintage range in natural language. The agent searches configured data sources (CoStar via MCP, internal comp database, uploaded OM PDFs), ranks matches by relevance, and populates a deal comp table in the Winston chat workspace — fully cited, with source links. User reviews and pins comps to the deal record.

**Which module:** Deal pipeline module + chat workspace + document ingestion pipeline. New MCP tools for CoStar or equivalent CRE data source. Leverages existing document pipeline for OM ingestion.

**Enterprise value:** Deal teams spend 4–8 hours per deal manually pulling and formatting comps. An agent that does this in under 5 minutes — with full source citations — directly compresses underwriting cycle time and reduces analyst overhead. A deal director can request comps mid-call and have them ready before the call ends.

**Implementation complexity:** Medium — chat workspace and document pipeline are existing infrastructure. New MCP tools needed for external CRE data sources (CoStar API or similar). Comp ranking logic is new but tractable.

**Priority score:** 8

**Priority rationale:** Deal Radar is already a Winston module; this is a direct capability upgrade, not a greenfield build. Agentic comp sourcing is a concrete, demo-able workflow that resonates immediately with asset managers and deal directors. Competitive with purpose-built CRE deal tools.

### 4. AI Governance Audit Trail for Winston Responses

**Market signal:** Gartner's AI maturity benchmark shows that organizations without evidence-quality audit trails trail by 20–32 points on every AI governance metric. Enterprise AI buyers now require: user identity, source data references, model version, prompt context, outputs, approvals, and downstream actions — all captured and queryable. "Evidence-first" is the 2026 governance posture. *(SecurePrivacy, Kiteworks, Larridin — 2026)*

**Feature idea:** Winston Response Provenance — each AI response block in the chat workspace displays an expandable "source chain": which MCP tools were called, which documents or data rows were retrieved, what model version produced the output, and a unique response ID. Responses are stored with full provenance in a `response_audit` table. Admins can export provenance logs for compliance review.

**Which module:** `backend/app/services/ai_gateway.py` + `ai_conversations.py` + `assistant_blocks.py` — extend the existing SSE event pipeline to emit provenance metadata. Frontend response block component gets an expandable "sources" section.

**Enterprise value:** When a CFO acts on a Winston-generated IRR analysis, they need to know it was grounded in the actual rent roll, not a hallucination. Provenance turns Winston into a defensible analytical tool — not just a chatbot. Essential for audit, LP queries, and internal sign-off workflows.

**Implementation complexity:** Medium — the AI gateway already tracks tool calls via SSE events. Capturing and persisting them as structured provenance is an extension of existing plumbing. The response block UI needs a new expandable section.

**Priority score:** 7

**Priority rationale:** Addresses the same enterprise governance requirement as feature #2 but from the response side rather than the tool side. Important for credibility but slightly less urgent than MCP audit trail because it's less visible to procurement decision-makers. Still high-value for day-to-day analyst trust.

### 5. LP Portal — Self-Serve Performance Access

**Market signal:** LP expectations are hardening: transparency, speed, and "proof of digital competence" are now central to fundraising and ongoing investor communication. Venture capital invested $1.7B in proptech in January 2026 alone — a 176% YoY increase — with durable, LP-facing platforms attracting premium valuations. Purpose-built LP portal competitors (Juniper Square, iLEVEL) are marketing self-serve dashboards as a differentiator. *(PYMNTS, Commercial Observer, CNBC — 2026)*

**Feature idea:** Winston LP Portal — a read-only, authenticated view at `/portal/[lpSlug]` where LPs can log in (magic link or SSO) and view their fund exposure, distributions history, capital account summary, and the last N quarterly reports. No write access. Data sourced from Winston's existing fund/LP schema. Optionally white-labeled per firm.

**Which module:** New frontend surface in repo-b + new portal auth flow (separate from firm-internal auth). Reads from existing LP, fund, and waterfall tables. New `lp_portal_sessions` table for access logging.

**Enterprise value:** Eliminates the quarterly "can you send me my K-1 and the latest fund update?" LP email. GPs spend 1–2 days per quarter answering LP inquiries that could be self-served. A portal also signals institutional-grade operations during LP due diligence.

**Implementation complexity:** High — new auth surface, new route segment, white-label theming, and read-only data access controls are all net new. Data model is largely in place but access control for LP-specific views requires careful scoping.

**Priority score:** 7

**Priority rationale:** High enterprise value and competitive necessity, but high build complexity earns a slightly lower immediate priority than the workflow-level features. This is a 2–3 sprint effort; the governance and agentic features above are 1-sprint bets with faster ROI for current customers.

### 6. Scenario Modeling Assistant — Natural Language Sensitivity Analysis

**Market signal:** Agentic AI systems in 2026 are moving toward "goal-driven operators" — not just answering questions but running multi-step analyses and iterating. PE firms are using AI to identify iteration opportunities in forecast models. Multi-agent orchestration is enabling complex financial modeling workflows previously requiring an analyst. *(Deloitte, EY, Stack AI — 2026)*

**Feature idea:** Winston Scenario Assistant — a chat workspace command that lets a user say "show me IRR sensitivity to exit cap rate from 5.5% to 7.0% in 25bps steps" and receive an inline sensitivity table + line chart rendered as a response block. The agent parameterizes Winston's existing scenario modeling engine, runs the grid, and renders results inline. Users can pin scenarios to a deal record.

**Which module:** Scenario modeling module + chat workspace + response block rendering (inline charts). This feature leverages the group-by/multi-series data work already in progress.

**Enterprise value:** Underwriting sensitivity analysis is a daily task for REPE asset managers. Currently this requires a spreadsheet — Winston can own this workflow natively, reducing context switching and keeping analysis inside the platform's audit trail.

**Implementation complexity:** Low-Medium — scenario engine exists; the incremental work is (a) natural language parameter parsing via the AI gateway and (b) rendering the output grid as an inline chart block. Builds directly on the response block work in the current sprint.

**Priority score:** 6

**Priority rationale:** The underlying engine is already built. This feature is essentially a natural language interface bolted onto existing infrastructure. Lower priority only because the chat workspace and response blocks must ship first — this is a fast follow once those land. Score would jump to 9 the moment those are done.

### 7. Multi-Agent Deal Underwriting Pipeline

**Market signal:** Multi-agent AI systems are the defining architecture of 2026 enterprise AI — orchestrators delegating to specialist sub-agents, with each agent handling a distinct task and passing results forward. Gartner predicts 40% of enterprise apps will feature task-specific agents by end of 2026. Deloitte calls this the shift to a "silicon-based workforce." *(Gartner, Deloitte, DEV Community — 2026)*

**Feature idea:** Winston UW Pipeline — a multi-agent orchestration layer triggered when a new deal enters the pipeline. Orchestrator delegates to: (1) a document ingestion agent that parses the OM and rent roll; (2) a market data agent that pulls comps from configured sources; (3) a financial modeling agent that populates the UW template with extracted inputs; (4) a risk flagging agent that checks assumptions against the firm's underwriting policy. Output is a pre-populated UW model with a "confidence score" per assumption and flagged exceptions for human review.

**Which module:** Deal pipeline + document ingestion pipeline + MCP tools + AI gateway. New orchestration layer in the backend. Significant new coordination logic.

**Enterprise value:** A full deal underwriting takes 1–2 weeks of analyst time. A Winston pipeline that delivers a 70% complete UW model in 30 minutes — with flagged assumptions for review — compresses deal velocity dramatically and lets senior staff focus on judgment rather than data entry.

**Implementation complexity:** High — requires multi-agent orchestration, reliable document extraction at scale, and assumption validation logic. This is a flagship feature, not a quick win.

**Priority score:** 6

**Priority rationale:** Enormous enterprise value but high complexity and dependencies on several upstream features (document pipeline, comp sourcing, scenario modeling) that should land first. This is the 6-month roadmap north star, not the next sprint. Priority score reflects sequencing, not importance.
