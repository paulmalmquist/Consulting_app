## New Ideas (Beyond Previous Radar)

### 1. ENHANCEMENT — Anthropic Marketplace Distribution Channel for Winston

**Market signal:** Anthropic launched an Enterprise Marketplace allowing enterprise customers to purchase third-party applications built on Claude using existing budget commitments. Launch partners include Snowflake, Harvey, and Replit. This creates a new B2B distribution channel specifically for Claude-native applications.

**Per CAPABILITY_INVENTORY.md, Winston already has:** Full Claude-native platform (AI gateway, 31 MCP tool categories, RAG, assistant blocks), 32 lab environment types for demos, and existing sales intelligence pipeline.

**What's missing:** Winston has no marketplace listing or packaging strategy for the Anthropic Enterprise Marketplace. This isn't a feature build — it's a go-to-market enhancement. Winston needs: (a) a marketplace-ready product description and pricing tier, (b) a streamlined onboarding flow that works within the marketplace's provisioning model, and (c) positioning copy that leverages Winston's Claude-native architecture as a first-mover advantage on the marketplace. The 2026-03-21 morning radar noted the Anthropic/Blackstone narrative (Idea #3) but did NOT identify the marketplace as a distribution channel.

**How it differs from existing radar:** The 2026-03-21 radar Idea #3 was about *narrative positioning* ("Winston is what Anthropic is pitching Blackstone"). This is about *distribution* — getting Winston listed where Anthropic enterprise customers already have budget. Different workstream, different owner (operations vs. marketing copy).

**Classification:** ENHANCEMENT (go-to-market, not code)
**Signal strength:** 5/5 — Direct distribution channel for Claude-native apps with enterprise buyers
**Complexity:** 2/5 — Packaging, listing copy, and onboarding flow; no new product code
**Differentiation:** 5/5 — First REPE platform on the Anthropic Marketplace

### 2. ENHANCEMENT — MCP Tool Marketplace / Plugin Architecture Inspired by NVIDIA Agent Toolkit Adoption

**Market signal:** NVIDIA's Agent Toolkit at GTC 2026 signed 17 major enterprise adopters (Adobe, Salesforce, SAP, ServiceNow) by offering an open platform where partners build domain-specific agents on shared infrastructure. The key insight isn't the toolkit itself (SKIPped in 2026-03-21 radar) — it's the *adoption model*: enterprises want to extend agent platforms with their own domain tools, not just consume pre-built ones.

**Per CAPABILITY_INVENTORY.md, Winston already has:** MCP registry with 31 tool categories, tool schemas, permissions, and audit policy. This is already a rich tool platform.

**What's missing:** Winston's MCP tools are all first-party (built by Novendor). There's no mechanism for a client firm to register their own custom MCP tools — e.g., a firm's proprietary valuation model, their internal compliance checker, or a custom data connector to their data warehouse. Adding a "bring your own tool" registration flow to the MCP registry would transform Winston from a closed tool platform to an extensible one. This directly mirrors the NVIDIA pattern: platform + partner tools = stickiness.

**Which module:** Enhancement to `backend/app/mcp/` (tool registration API), new admin UI in MCP management pages, schema validation for third-party tool definitions.

**Classification:** ENHANCEMENT
**Signal strength:** 4/5 — NVIDIA's 17-partner adoption validates the extensibility model
**Complexity:** 3/5 — MCP registry exists; needs registration flow, validation, and sandboxing
**Differentiation:** 5/5 — No REPE competitor offers client-extensible AI tool registries

**Priority score:** 6
