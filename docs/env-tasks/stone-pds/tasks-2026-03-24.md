## Priority Queue (for today's coding session)

### P0 — Critical (fix immediately)

**1. Fix Resources page null guard** (`/pds/resources`)
- Error: `TypeError: Cannot read properties of undefined (reading 'length')` at line ~3340 in compiled bundle
- Root cause: `heatmapMonths` state arrives as `undefined` instead of `[]` when the `/api/pds/v2/utilization/heatmap` API returns a null/empty `months` field. The `setHeatmapMonths(hmRes.months)` call doesn't guard against undefined.
- Fix: Change `setHeatmapMonths(hmRes.months)` → `setHeatmapMonths(hmRes.months ?? [])` and `setHeatmap(hmRes.rows ?? [])`. Also add `setCapacityDemand(cdRes.rows ?? [])`.
- File: `repo-b/src/app/lab/env/[envId]/pds/resources/page.tsx` — lines ~70–80 (fetchData callback, setHeatmapMonths/setHeatmap assignments)

**2. Fix Timecards page null guard** (`/pds/timecards`)
- Error: `z is not iterable` — crash at `[...bench].sort(...)` when `bench` state is undefined
- Root cause: `setBench(benchRes.employees)` — if `benchRes.employees` is undefined, `bench` state becomes undefined instead of `[]`. The spread `[...bench]` then throws.
- Fix: `setBench(benchRes.employees ?? [])`, `setDistribution(distRes.bins ?? [])`, `setOvertime(distRes.overtime_alerts ?? [])`.
- File: `repo-b/src/app/lab/env/[envId]/pds/timecards/page.tsx` — lines ~73–76 (state assignments in fetchData)

**3. Fix Client Satisfaction page null guard** (`/pds/satisfaction`)
- Error: `TypeError: Cannot read properties of undefined (reading 'length')` — same pattern as Resources
- Root cause: One or more API responses returning undefined arrays; component calls `.length` before null check
- Fix: Apply `?? []` guards on all state assignments after API fetch in the satisfaction page fetchData callback
- File: `repo-b/src/app/lab/env/[envId]/pds/satisfaction/page.tsx` — fetchData callback, all array state setters
