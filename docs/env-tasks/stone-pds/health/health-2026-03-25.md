## New Issues Found Today

### 1. NPS Score "NaN" on Client Satisfaction page — MEDIUM
- **What**: NPS Score KPI card renders "NaN" instead of a numeric score
- **Where**: `/pds/satisfaction` — NPS Score summary card, top of page
- **Likely cause**: NPS calculation (`promoters - detractors / total_responses * 100`) dividing by 0 or operating on undefined. The seed data has scores but the NPS `pds_nps_responses` table may not have the right promoter/detractor distribution computed, or the API is returning null for the computed NPS field.
- **Not a crash** — page is stable, verbatims load, trend panel shows "No trend data available"
- **Fix path**: Check `GET /api/pds/v2/satisfaction/nps-summary` response for `nps_score` field; verify it's not null/undefined before rendering

### 2. Forecast "Total Deals: NaN" — MEDIUM
- **What**: Deal count renders "NaN" on the Forecast/Pipeline page
- **Where**: `/pds/forecast` — Deal Funnel chart, "Total Deals: NaN" label
- **Likely cause**: `deals.length` or similar aggregation called on null/undefined array, or a numeric conversion of an empty string. Chart renders with `$0k–$340k` axis and status breakdown, so data partially loads.
- **Not a crash** — page is otherwise functional, Weighted Pipeline $523,990 renders correctly
- **Fix path**: Check `pds_pipeline_deals` or equivalent table for the Stone env; guard `deals?.length ?? 0` in the chart label computation
