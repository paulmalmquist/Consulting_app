## What Was Done

### P0: Fix Tech Adoption page crash (`/pds/adoption`)

**Working on:** Tech Adoption crash because it's the only P0 (page completely crashes with error boundary) and blocks demo navigation. `[source: planning/2026-03-26.md, demo-friction-log.md]`
**Confidence this was the right priority:** HIGH

**Root cause:** `fetchData` callback in `adoption/page.tsx` lines 102-105 set state directly from API response properties (`overviewRes.tools`, `healthRes.accounts`, `trendsRes.rows`, `trendsRes.tool_names`) without null guards. When the API returns `undefined` for any of these, the component attempts to iterate/spread on `undefined`, causing "_ is not iterable".

**Fix:** Added `?? []` null coalescing on all four state setters:
- `setTools(overviewRes.tools ?? [])`
- `setHealthScores(healthRes.accounts ?? [])`
- `setTrends(trendsRes.rows ?? [])`
- `setTrendToolNames(trendsRes.tool_names ?? [])`

**File changed:** `repo-b/src/app/lab/env/[envId]/pds/adoption/page.tsx` (lines 102-105)

### P1: Fix Forecast "Total Deals: NaN" (`/pds/forecast`)

**Root cause:** Two issues:
1. `setPipeline(res.stages)` and `setPortfolio(res.accounts)` had no null guards (same pattern as adoption)
2. The summary strip `pipeline.reduce((s, r) => s + r.count, 0)` produces NaN if any `r.count` is null/undefined

**Fix:**
- Added `?? []` on `setPipeline` and `setPortfolio`
- Changed reduce to `s + (r.count ?? 0)` and `s + (r.weighted_value ?? 0)`

**File changed:** `repo-b/src/app/lab/env/[envId]/pds/forecast/page.tsx` (lines 67, 70, 192, 197)

### P1: Fix NPS Score NaN on Client Satisfaction (`/pds/satisfaction`)

**Root cause:** `npsSummary.current_nps` passed to `<NpsGauge score={...}>` can be null/NaN from the API. Also `response_count` and `response_rate_pct` can be null.

**Fix:**
- Guarded `current_nps` with `!= null && !isNaN()` check, defaulting to 0
- Guarded `response_count` with `?? 0`
- Guarded `response_rate_pct` with `?? 0` in state and NaN-safe display in JSX
- The existing `trend: npsRes.trend ?? []` guard was already present (prior fix)

**File changed:** `repo-b/src/app/lab/env/[envId]/pds/satisfaction/page.tsx` (lines 109-115, 165-166)
