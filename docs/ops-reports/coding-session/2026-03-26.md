## What Was Done

### 1. NPS Score NaN Fix (Stone PDS Satisfaction Page)
**Root cause:** Backend `get_nps_summary()` returned `{"nps_quarterly": [...]}` but the frontend expected `{current_nps, trend, response_count, response_rate_pct}`. The cast to `NpsSummary` left `current_nps` as undefined, and `Math.round(undefined)` produced NaN.

**Fix:** Backend now computes and returns `current_nps` (most recent quarter), `trend` (quarter/nps pairs), `response_count` (total), and `response_rate_pct` (placeholder 100). Keeps `nps_quarterly` for backwards compatibility. Also added `Number.isFinite()` guard in `NpsGauge.tsx` so any future null/undefined score renders as 0 instead of NaN.

**Files changed:**
- `backend/app/services/pds_satisfaction_analytics.py` — reshaped return value
- `repo-b/src/components/pds-enterprise/NpsGauge.tsx` — defensive guard

### 2. Total Deals NaN Fix (Stone PDS Forecast Page)
**Root cause:** Backend pipeline stages returned `deal_count` and `total_value` but the frontend PipelineStage type expected `count` and `unweighted_value`. Accessing `r.count` returned undefined, and summing undefined values produced NaN.

**Fix:** Backend now includes both `count` (matching frontend) and `deal_count` (backwards compat), plus `unweighted_value` (matching frontend) and `total_value` (backwards compat).

**Files changed:**
- `backend/app/services/pds_revenue_analytics.py` — added aliased fields

### 3. Schedule Health Page (P1 Carry-Forward)
**Root cause:** `/pds/schedule` silently redirected to `/pds/risk`, which was confusing when users navigated via the sidebar.

**Fix:** Replaced the redirect with a proper Schedule Health page that shows SPI (Schedule Performance Index) metrics, KPI cards, a horizontal bar chart of SPI by project, and a project table sorted by schedule risk. Uses the existing `/api/pds/v2/analytics/portfolio-health` endpoint.

**Known limitation:** Page uses `worst_10` data from portfolio-health, so SPI metrics are based on the 10 worst-performing projects, not the full portfolio.

**Files changed:**
- `repo-b/src/app/lab/env/[envId]/pds/schedule/page.tsx` — full rewrite from redirect to real page
