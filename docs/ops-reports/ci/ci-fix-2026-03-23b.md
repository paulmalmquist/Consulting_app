## Root Cause

The `commitsync` commit (`8b06eee4`) on 2026-03-23 consolidated formatting utilities into `@/lib/format-utils.ts`. As part of this migration, it:

1. Created `repo-b/src/lib/format-utils.ts` with `fmtNumber(v: number | string | null | undefined): string` — **only 1 parameter**
2. Removed the local `fmtNumber(value: unknown, digits = 1): string` from `SustainabilityWorkspace.tsx` — which accepted `unknown` and had a second `digits` parameter
3. Added an import of the canonical `fmtNumber` from `@/lib/format-utils`

This caused two classes of CI errors:
- **TS2554** ("Expected 1 arguments, but got 2"): All `fmtNumber(x, 0)` / `fmtNumber(x, 1)` / `fmtNumber(x, 4)` call sites passed a second decimal precision argument
- **TS2345** ("Argument of type 'unknown' is not assignable"): Call sites that pass values from `Record<string, unknown>` (e.g., `assetDashboard.trends.monthly_energy`) couldn't satisfy the new stricter type

Note: These errors only manifested in CI (not locally) because the local `node_modules` was missing `next`'s type declarations, so TypeScript resolved fewer errors.

---

## Fix Applied

**Commit 1** (`ab44135d`): `SustainabilityWorkspace.tsx` — Added thin local wrapper functions that accept `unknown` and delegate to the canonical functions:
```typescript
function fmtNumber(value: unknown, digits = 1): string {
  return _fmtNumber(value as number | string | null | undefined, digits);
}
```

**Commit 2** (`a974bcf2`): `format-utils.ts` — Added the missing `decimals?: number` optional parameter to `fmtNumber`:
```typescript
export function fmtNumber(v: number | string | null | undefined, decimals?: number): string
```

Both fixes together resolved the CI failures. The canonical `format-utils.ts` now correctly accepts an optional decimals argument (matching what all call sites expect), and `SustainabilityWorkspace.tsx` has type-safe wrappers for the `unknown`-typed call sites.
