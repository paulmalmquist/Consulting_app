## Improvement Recommendations

### Priority 1: CRITICAL — Rotate Exposed API Key
`openaikey.txt` exists at repo root with a live OpenAI API key (`sk-proj-F6QN...`). While `.gitignore` lists it, the file is present in the working directory. Database credentials are also in `.env.local` files. **Action:** Rotate all exposed credentials immediately. Ensure `openaikey.txt` is not in git history (check with `git log --all -- openaikey.txt`).

### Priority 2: HIGH — Fix the Build-Then-Fix Loop
37 of 76 commits (49%) are fixes for issues that should have been caught before committing. **Action:** Add pre-commit hooks:
- `ruff check --fix` for Python lint
- `tsc --noEmit` for TypeScript type checking
- `pytest tests/ -x --timeout=30` for quick test validation
This alone would eliminate ~15 fix commits per week.

### Priority 3: HIGH — Address Bug 0 Before New Features
Bug 0 (execution narration / tool call spam) is marked CRITICAL in LATEST.md but received zero commits this week. Meanwhile, 5+ new AI-adjacent features were built. **Action:** The autonomous coding agent's priority ordering should enforce: "Fix CRITICAL bugs before building new features."

### Priority 4: HIGH — Fix AI Feature Testing Infrastructure
Chrome extension automation has been broken for 2+ consecutive days (Mar 19-20), meaning zero AI regression testing during a week when AI gateway code was modified. **Action:** Either fix the Chrome extension connectivity or implement a fallback test approach (API-level testing that doesn't require browser automation).

### Priority 5: HIGH — Test Coverage for Critical Services
The 5 largest backend services (ai_gateway.py: 3,947 lines, pds.py: 3,083 lines, winston_demo.py: 2,230 lines, re_sustainability.py: 2,026 lines, pds_enterprise.py: 1,894 lines) have ZERO dedicated tests. Overall backend test coverage is ~8% of services. **Action:** Prioritize tests for ai_gateway.py and credit_decisioning.py as they are the core revenue-generating features.

### Priority 6: MEDIUM — Enforce Commit Discipline
Three commits this week have vague or useless messages: `"green"`, `"deploy2"`, `"aesthetic"`. One commit (`3e98d88`) is a kitchen-sink with 5+ unrelated changes. **Action:** Add commit message linting (commitlint or similar). Enforce atomic commits in the coding agent's instructions.

### Priority 7: MEDIUM — Extract Duplicated Utility Functions
Multiple `re_*` services duplicate `_d()`, `_q()`, `_canonical_json()`, `_compute_hash()`, and scenario comparison logic. **Action:** Create `backend/app/services/_re_utils.py` and `_finance_utils.py` shared modules.

### Priority 8: MEDIUM — Activate Ops Tasks
5 scheduled ops tasks (nightly-ops-validator, deploy-smoke-test, morning-ops-digest, midday-production-health, novendor-market-scanner) have NEVER produced output. Their directories are empty. **Action:** Investigate why these tasks aren't running. Either fix their execution or remove them from LATEST.md to avoid a false sense of coverage.

### Priority 9: LOW — Refactor Oversized Services
`ai_gateway.py` at 3,947 lines violates any reasonable file size guideline. **Action:** Split into at least 3 modules: `ai_gateway_routing.py`, `ai_gateway_rag.py`, `ai_gateway_conversation.py`.

### Priority 10: LOW — Clean Up v1/v2 Route Pairs
4 route files have v2 variants alongside v1 originals. **Action:** If v2 is production, deprecate v1. If both are needed, document why in comments.
