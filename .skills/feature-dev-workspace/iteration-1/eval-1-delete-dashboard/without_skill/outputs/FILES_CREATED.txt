RESPONSE CODES:
===============
200 OPTIONS    - Header: Allow: DELETE, OPTIONS
204 DELETE     - Success (No Content)
404 DELETE     - Dashboard not found
503 DELETE     - Database unavailable
500 DELETE     - Database error

KEY DESIGN DECISIONS:
=====================
1. 204 No Content: Standard REST pattern for successful DELETE
2. Cascade Delete: Uses PostgreSQL foreign key constraints (no app-level logic)
3. Simple Query: Single DELETE statement (unlike complex funds deletion)
4. Parameterized Queries: SQL injection prevention
5. Two-stage delete: Verify existence first, then delete (explicit 404 handling)

ERROR HANDLING:
===============
Database Unavailable (503)
  - getPool() returns null
  - Response: { "error": "Database unavailable" }

Dashboard Not Found (404)
  - Initial SELECT query returns 0 rows
  - Response: { "error": "Dashboard not found" }

Database Error (500)
  - Any exception during query execution
  - Logged to console: "[dashboards/delete] Error: ..."
  - Response: { "error": "Failed to delete dashboard" }
